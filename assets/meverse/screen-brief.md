# Meverse — Screen Brief v0.2

Meverse v0.2 — Stress test
Full pipeline live — shortcut → n8n → Google Sheets
Automated capture, validated end to end

**Date:** 2026-03-06
**Status:** First draft — not validated by users yet
**Based on:** Hand-drawn wireframe 2026-03-06 + pressure test Session 13

---

## Screen 1 — Onboarding

One binary choice. Smile or frown. That's the entire setup.
No feature explanation, no carousel, no promises.
The first tap IS the onboarding — the product starts the moment
you make your first capture.

Optional depth available immediately after: percentage level (0-10)
and a comment field for context. Both optional. Neither required.
Progressive disclosure — the user goes as deep as they want,
never deeper than they're ready for.

Must never do: explain the product before the user has felt it.

## Screen 2 — Widget Lock Screen

A four-point capture widget on the lock screen.
Center `+` is the primary action — tap once, capture begins.
Four surrounding buttons: task, text, voice, camera.
These are input modalities, not states.

Capture starts immediately on tap. Every character recorded
without manual save. If no action taken in 90 seconds —
signal auto-sends as partial capture.

The only interruptions: Cancel or Save.
Classification happens later, never at capture time.

Must never do: ask for confirmation, delay with animation,
or interrupt the moment that caused the trigger.

## Screen 3 — Chat / Retrieve

The lazy view. A chronological feed of your own captures —
formatted as a private conversation with yourself.

Primary use: quick retrieval, recent history, unstructured review.
Not the only view — graphs and structured analysis come later.

Must never do: look like a social feed. No likes, no reactions,
no performance layer. This is a mirror, not a stage.

## Screen 4 — Statistics

A calendar heatmap. Three colors. One year.

```
Green  = state 1 — Life
Amber  = state 0 — Poke
Red    = state 2 — Upset
Grey   = no signal that day
```

Three months of inner weather visible in five seconds.
No interpretation. No AI commentary. Observation only.
The shape of your year — not a diagnosis of it.

Sidebar: New Input, Projects, Stats.
Graph builder is future — Stage 9, earned by data.

Must never do: interpret prematurely. No "you seem stressed"
notifications until the data and the architecture justify it.

---

## What this is not

A journal. A mood tracker. A productivity app.
_"A living record of who you actually are."_

# Meverse — Screen Brief v0.3

Meverse v0.3 — Visual identity
8-bit pixel art, six canonical screens
The product has a face now

**Date:** 2026-03-06
**Status:** First visual identity — not validated by users yet
**Based on:** 8-bit pixel art generation, six canonical screens

---

## Visual language

Pixel RPG aesthetic. Ancient temple. Crescent moon. Gold-framed
ornamental widget. The ledger as a magical artifact.
The user as the hero entering the gate.
The Chaos Magic thesis made visible.

## Screen 1 — Onboarding

Ancient temple gate. Treasure chest above the arch.
Wizard figure beside the Register button.
The door is open. The product does not explain itself.
Register or enter. Nothing else.

## Screen 2 — Widget Lock Screen

Gold-framed four-point widget centered on a dark background.
Center `+` glowing. Four modalities: task, text, voice, camera.
Constellation top left. Weather top right.
The gate looks like a gate.

## Screen 3 — Auth Flow

Phone or email toggle. Social auth: Apple, Meta, Google.
Dark background. Pixel border. Terms visible but unobtrusive.
Clean entry, no friction.

## Screen 4 — Self-messages

Dark list view. Chronological captures as pill-shaped cards.
Checkboxes for completed items.
Type a note field pinned to the bottom.
No feed mechanics. No reactions. A mirror.

## Screen 5 — Statistics

Fractal Map View. AI Notes Analysis.
1024 unique data connections rendered as a color-coded
geometric node graph. Graph, Folders, Hierarchy navigation.
This is Stage 8 as a vision artifact —
earned by data, not deployed yet.
