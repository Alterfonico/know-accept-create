# ADR-003: System Roadmap

**Date:** 2026-03-02
**Status:** Accepted
**Domain:** System Architecture

---

## The Through-Line

One axiom. One KW. Three questions closing every cycle.
No stage begins until the previous one is witnessed by reality.

---

## The Map (so far)

```
Stage 0   Foundation
          Axiom. KW. Repository. Protocol.
          ✓ Complete

Stage 1   Zone A — Nursery Gate
          Trinary input. iPhone shortcut.
          Telegram as proof of life channel.
          ↑ Current — state variable fix pending

Stage 2   Zone B — Processing Layer
          n8n webhook receives the signal.
          First automation logic. Routes to Zone C.

Stage 3   Zone C — Tally Ledger
          Google Sheets as database.
          Longitudinal pattern storage begins.
          The system starts remembering.

Stage 4   Zone A Upgrade — Body-First Capture
          Smartwatch gesture input.
          Built from three months of friction data.

Stage 5   Pattern Recognition
          Human eye analysis first.
          No algorithm until the human sees it clearly.

Stage 6   Schema Upgrade
          Real database designed from actual data.
          Not assumptions — evidence.

Stage 7   KW as Agent
          Axiom validator running automatically.
          Drift detection across all zones.

Stage 8   Twin Pairs Born from Data
          First differentiation signal issued by KW.
          Earned by three cycles of systematic deviation.

Stage 9   Data Driven Diagnosis
          Patterns become findings.
          The system surfaces what the student couldn't see alone.

Stage 10  Data Driven Prescription
          Personalized interventions from pattern history.
          Not generic advice — earned by the data.

Stage 11  Open Source Gift
          One person's inner work becomes
          infrastructure for others on the same road.
```

---

## The Rule

Each stage closes with three questions before the next opens.
The KW witnesses every transition.
No stage is skipped because the data didn't justify it.
