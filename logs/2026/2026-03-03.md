## Entry 06 — Deduplication Patch

### Surgical Fix to ingest-thought

**Trigger:** Diagnosed root cause of duplicates. Wrote and deployed fix.

**Context:** Added a deduplication check querying `metadata->>slack_ts`
before any expensive processing. If a row with the same Slack timestamp
already exists, the function exits immediately and does nothing.

**The patch — inserted after the empty message guard:**

```typescript
// Skip if this Slack message was already ingested (deduplication via slack_ts)
const { data: existing } = await supabase
  .from("thoughts")
  .select("id")
  .eq("metadata->>slack_ts", messageTs)
  .limit(1);
if (existing && existing.length > 0) {
  console.log(`Duplicate event for slack_ts ${messageTs}, skipping.`);
  return new Response("ok", { status: 200 });
}
```

**Observation:** Connector deploy failed twice on Supabase's end (internal
error). Patch was applied manually via the dashboard instead. No refactor
needed — the existing code was clean and the addition was minimal.

**Reminder:** Never validate a patch without fully understanding each line,
or without professional supervision.
